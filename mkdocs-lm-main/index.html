Instalación de MkDocs

Para instalar MkDocs, sigue estos pasos:

  1. Asegúrate de tener Python 3 instalado en tu sistema. Si no lo tienes, descarga e instala Python 3 desde el sitio web oficial de Python.

  2. Abre la línea de comandos en tu sistema operativo.

  3. Ejecuta el siguiente comando para instalar MkDocs y sus dependencias:

pip install mkdocs

Si tienes problemas para instalar MkDocs debido a permisos de usuario, puedes agregar el parámetro --user al final del comando para instalarlo solo para el usuario actual:

pip install mkdocs --user
  1. Una vez que MkDocs esté instalado, puedes verificar la versión instalada con el siguiente comando:

mkdocs --version

Con estos pasos, tendrás MkDocs instalado en tu sistema y podrás comenzar a utilizarlo para crear documentación estática para tus proyectos.

Comandos

MkDocs es una herramienta de línea de comandos que permite crear documentación estática utilizando archivos Markdown y un conjunto de plantillas de sitio web personalizables. A continuación se detallan algunos de los comandos más utilizados en MkDocs:

  1. mkdocs new <nombre>: Crea un nuevo proyecto de documentación con el nombre especificado.
  2. mkdocs serve: Inicia un servidor web local que muestra la documentación generada. Los cambios realizados en los archivos Markdown se actualizarán automáticamente en el servidor local.
  3. mkdocs build: Genera la documentación estática en la carpeta "site" del proyecto. Esto debe hacerse antes de publicar la documentación en un servidor web.
  4. mkdocs gh-deploy: Publica la documentación generada en GitHub Pages.
  5. mkdocs pdf: Genera un archivo PDF a partir de la documentación. Se requiere la instalación de una herramienta adicional llamada WeasyPrint para utilizar este comando.
  6. mkdocs --version: Muestra la versión actual de MkDocs instalada en el sistema.

Estos son solo algunos de los comandos más utilizados en MkDocs. Hay muchos otros comandos y opciones disponibles que se pueden consultar en la documentación oficial de MkDocs.

Crear una página web en MkDocs

Para crear una página web con MkDocs, sigue estos pasos:

  1. Crea un nuevo proyecto de documentación ejecutando el siguiente comando en la línea de comandos:
mkdocs new my-docs

Este comando creará una nueva carpeta llamada "my-docs" que contendrá la estructura básica de un proyecto de documentación de MkDocs.

Cambia al directorio del proyecto recién creado con el siguiente comando:

cd my-docs

Abre la carpeta "docs" que se encuentra dentro del proyecto. Allí encontrarás un archivo llamado "index.md". Este archivo es la página principal de tu documentación. Puedes editar este archivo utilizando Markdown para agregar contenido a la página.

Ejecuta el siguiente comando para iniciar un servidor web local y ver tu documentación en el navegador:

mkdocs serve

Esto abrirá un servidor local en el puerto 8000 que mostrará tu documentación. Para ver la documentación, abre un navegador y navega a la dirección http://localhost:8000.

Para personalizar la apariencia de tu documentación, puedes modificar el archivo "mkdocs.yml" que se encuentra en la raíz del proyecto. Este archivo contiene opciones de configuración para la plantilla de sitio web que se utiliza para generar la documentación.

Una vez que estés satisfecho con la apariencia y el contenido de tu documentación, ejecuta el siguiente comando para generar la versión estática de la documentación:

  1. mkdocs build

Esto generará una carpeta llamada "site" que contendrá la versión estática de tu documentación.

  1. Finalmente, puedes publicar la documentación en un servidor web utilizando cualquier método que desees. Por ejemplo, puedes subir la carpeta "site" a un servidor web estático como GitHub Pages o AWS S3.

Con estos pasos, tendrás una página web básica creada con MkDocs que puedes personalizar y publicar en línea.
